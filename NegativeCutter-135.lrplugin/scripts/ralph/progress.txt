## Codebase Patterns
- Use `Image.Image` type annotation for PIL image variables to avoid mypy `Incompatible types in assignment` errors when reassigning from `Image.open()` to `.convert()` results.
- Use explicit `Optional[str]` (or `str | None`) for function parameters with `None` defaults; mypy defaults to `no_implicit_optional=True`.
- The filmcrop package already contains detector.py, export.py, api.py, and gui/ ŌĆö later stories build on this foundation.
- `python -m filmcrop.detector --help` requires an `if __name__ == "__main__":` block in detector.py (it is a module, not a package).
- Legacy CLI compatibility is maintained via `detect_thumb.py` which imports from `filmcrop.detector`.
- When using `QTimer.singleShot` with a lambda that calls a method, type narrowing from an outer scope check (e.g. `if not self._image_path: return`) does not propagate into the delayed method. Add an explicit `None` guard inside the delayed method.
- `psutil` is an optional dependency throughout the project; always guard with `try/except ImportError` and set a `HAS_PSUTIL` flag.
- Use `QMessageBox.critical(parent, title, message)` for error dialogs in the GUI; the status bar alone is insufficient for exceptions.
- `ImageView.update_frame_geometry()` must be called after any frame coordinate change to keep both the rect overlay and its label text position in sync.
- After programmatically modifying frame coordinates (e.g., min-size enforcement in `_on_spinbox_changed`), call `_sync_spinboxes()` to keep the coordinate editor spinboxes consistent with the data model.
- `DraggableFrameItem.frame_data()` is the public accessor for the underlying frame dict; prefer it over accessing `._frame` directly.
- FastAPI is an optional dependency; always check `filmcrop.api.has_api()` before starting the server, and show a `QMessageBox` with install instructions if unavailable.
- To shut down a `uvicorn.Server` running in a background thread, set `server.should_exit = True` (do not rely on daemon thread cleanup alone).
- Use a `QTimer` polling loop to display cross-thread metrics (e.g., API request count) in the PyQt6 status bar.
